

### ✅ آموزش نحوه کار - حتما خوانده شود

```markdown
# به تیم فرانت‌اند خوش آمدید! 🎉

این مخزن (Repository) محل کار تیم فرانت‌اند است.  
همه اعضا روی کدهای خود کار می‌کنند، تغییرات را مدیریت می‌کنند و گزارش کار روزانه خود را ثبت می‌کنند.

---

## 📌 قوانین کلی تیم

1. **هیچ‌کس مستقیماً روی `main` کار نمی‌کند.**
2. هر فرد یک **برنچ شخصی** دارد و تمام تغییراتش را در آن انجام می‌دهد.
3. گزارش کار روزانه در پوشهٔ شخصی خودش در `reports/` ثبت می‌شود.
4. برای اضافه کردن تغییرات به `main`، باید یک **Pull Request (PR)** ایجاد شود و توسط تیم بررسی و تأیید شود.

---

## 🔧 مراحل شروع کار (فقط یک‌بار)

هر فرد باید یک بار این مراحل را انجام دهد تا برنچ شخصی خود را بسازد:

### 1. ساخت برنچ شخصی
```bash
git checkout -b نام-شما-branch
```

مثال (برای علی):
```bash
git checkout -b ali-branch
```

### 2. ارسال برنچ به گیتهاب
```bash
git push -u origin نام-شما-branch
```

> ✅ این کار فقط یک‌بار انجام می‌شود. بعد از آن، هر روز از همین برنچ استفاده کنید.

---

## 📁 ساختار پوشه‌ها

### ۱. پوشه کدها
- همه کدهای فرانت‌اند خود را در پوشهٔ شخصی خود در `src/` قرار دهید.
- اگر پوشه وجود ندارد، آن را بسازید.

مثال:
```
src/ali/
src/sara/
```

در این پوشه، فایل‌های HTML، CSS، JS، یا کامپوننت‌های React/Vue و... قرار می‌گیرد.

---

### ۲. گزارش کار روزانه
هر روز در پوشهٔ `reports/`، یک فایل جدید با تاریخ روز بسازید.

مسیر نمونه:
```
reports/ali/1403-05-11.md
reports/sara/1403-05-11.md
```

> ✅ فقط خودتان می‌توانید در پوشهٔ شخصی خود تغییر ایجاد کنید.

---

## 💻 کار روزانه – مراحل انجام تغییرات

1. مطمئن شوید در برنچ شخصی خود هستید:
   ```bash
   git checkout نام-شما-branch
   ```

2. تغییرات خود را اضافه کنید:
   ```bash
   git add .
   ```

3. تغییرات را کامیت کنید (توضیح واضح بنویسید):
   ```bash
   git commit -m "کامپوننت هدر و فوتر ایجاد شد"
   ```

4. تغییرات را به گیتهاب بفرستید:
   ```bash
   git push
   ```

---

## 🔄 ارسال تغییرات به main (Pull Request)

وقتی آماده‌اید تغییراتتان به نسخه اصلی اضافه شود:

1. به [صفحه گیتهاب این پروژه](https://github.com/نام-پروژه) بروید.
2. روی دکمه **"Pull Requests"** کلیک کنید.
3. دکمه **"New Pull Request"** را بزنید.
4. از `نام-شما-branch` به `main` مقایسه کنید.
5. عنوان و توضیحات واضح بنویسید (مثلاً: "افزودن صفحه لاگین و ثبت‌نام").
6. PR را ارسال کنید و منتظر بررسی تیم بمانید.

> ⚠️ فقط بعد از تأیید PR، تغییرات وارد `main` می‌شوند.

---

## 🧠 نکات مهم

- 🔐 پوشه‌های شخصی (`src/نام` و `reports/نام`) فقط توسط خود فرد قابل ویرایش است.
- 📝 پیام کامیت‌ها واضح و معنادار باشد (ترجیحاً فارسی یا انگلیسی ساده).
- 🔄 قبل از شروع کار، از آخرین تغییرات `main` مطمئن شوید:
  ```bash
  git pull origin main
  ```
